# 🚀 前端架构演进与实施计划

> **当前架构**：Next.js 16 + React 19 + Tailwind v4 + Turbopack，纯前端静态方案，无数据库。

---

## 🎯 为什么当前阶段不需要数据库？

1. **内容静态化优势**：知识库与工具通过 `features.ts` 静态配置驱动，Next.js 渲染速度极快，运维成本趋近于零。
2. **状态本地化**：个人使用场景下，进度、配置等状态完全可以存于 `localStorage`。
3. **安全与轻量**：无后端即无鉴权、注入、存储刷库等安全面，专注打磨 UI/UX。

## ✅ Phase 4：微型应用注册制 — App Registry（已完成）

**核心架构：**

```
src/apps/
├── registry.ts          # AppId 枚举 + AppMeta 类型 + resolveAppUrl()
└── campus/
    └── MyGPA/index.tsx  # 首个试点应用
```

**数据流：**
```
features.ts (url: "app://mygpa")
    → FeatureWindow 检测 app:// 前缀
    → resolveAppUrl() 查注册表
    → AppRenderer 懒加载对应组件
    → React.Suspense 渲染
```

**已上线应用：** `mygpa` — MyGPA 计算器

**扩展规则：**
1. 在 `src/apps/<category>/<AppName>/index.tsx` 创建组件
2. 在 `src/apps/registry.ts` 的 `APP_REGISTRY` 中注册
3. 在 `src/components/feature-window/AppRenderer.tsx` 的 `appComponents` 中加懒加载入口
4. 在 `src/data/features.ts` 对应 folder 中加 `url: "app://<id>"` 条目

## 🔲 Phase 5：应用扩展 + 富文本笔记（计划中）

### 5a. 应用注册表扩展

**目标：** 新增 2-3 个微型应用（如番茄钟、待办清单），验证 Registry 架构的多应用可行性。

**分类体系：**
```
src/apps/
├── campus/        # 校园工具（MyGPA、课表、成绩…）
├── productivity/  # 效率工具（番茄钟、待办、日记…）
├── creative/      # 创意工具（白板、配色…）
├── dev/           # 开发工具（JSON 格式化、正则…）
└── study/         # 学习工具（单词卡、公式速查…）
```

**增长阈值：**
- 3-10 个：保持手动注册，现有架构足够
- 10-20 个：引入 `manifest.ts` 自动发现机制
- 20+ 个：应用商店视图（搜索/分类/最近使用）

### 5b. 富文本笔记系统

**方案：混合架构** — 编辑器是微型应用，笔记内容是 Markdown 文件。

```
src/apps/productivity/NoteViewer/index.tsx    # 渲染器（微型应用）
src/content/notes/系统解剖学.md                # 内容文件
src/content/_meta.ts                          # 内容索引
```

**渲染管道：**
```
features.ts (url: "app://note-viewer?id=系统解剖学")
  → AppRenderer → NoteViewer → 读取 MD → react-markdown 渲染
```

**分阶段实施：**
1. 先用 `react-markdown` 做只读 Viewer
2. 验证通畅后升级为 `Tiptap` 块编辑器
