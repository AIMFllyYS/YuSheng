# 项目详细目录结构规范

本项目基于 Next.js 16 框架构建，遵循以下目录结构规范：

```text
AIMFlly/
├── app/                  # Next.js App Router 路由层（薄路由，只做声明）
│   ├── globals.css       # 全局样式文件
│   ├── layout.tsx        # 全局根布局（字体注入、SEO meta）
│   ├── page.tsx          # 首页路由 → 代理给 HomeClient
│   └── features/         # 功能页子路由（每个仅 5 行，委托给 FeaturePage）
│       ├── notes/page.tsx
│       ├── memory/page.tsx
│       ├── share/page.tsx
│       └── revelation/page.tsx
├── src/                  # 核心源代码目录（业务逻辑与组件，与路由层严格分离）
│   ├── apps/             # 微型应用注册中心（App Registry）
│   │   ├── registry.ts   # 统一注册表：AppId、AppMeta 类型定义与 resolveAppUrl()
│   │   └── campus/       # 按分类组织的微型应用
│   │       └── MyGPA/    # MyGPA 计算器
│   │           └── index.tsx
│   ├── components/       # 可复用组件目录
│   │   ├── effects/      # 视觉效果与动画组件（粒子、羽毛特效等）
│   │   ├── feature-window/ # 功能窗口系统（文件树导航 + 应用调度）
│   │   │   ├── FeatureWindow.tsx  # 主窗口（支持 iframe / app:// 两种渲染模式）
│   │   │   ├── AppRenderer.tsx    # 微型应用渲染适配器（懒加载 React 组件）
│   │   │   └── FeaturePage.tsx    # 独立路由页包装器
│   │   ├── home/         # 首页场景组件（JourneyOverlay、HomeClient 等）
│   │   └── shared/       # 全局通用 UI 组件库
│   ├── data/             # 静态数据与配置（功能树、页面配置、站点配置）
│   │   ├── features.ts   # FeatureItem 文件树数据（所有功能窗口内容的唯一数据源）
│   │   ├── featurePages.ts # 各功能页的标题、图标、渐变等元数据
│   │   ├── config.ts     # 全局配置
│   │   └── site.ts       # 站点元信息
│   └── hooks/            # 自定义 React Hooks
├── docs/                 # 项目文档目录
│   ├── 规范/             # 项目开发规范与演进计划（当前目录）
│   └── 单页HTML/         # 历史 HTML 工具（通过 features.ts 映射，iframe 渲染）
├── public/               # 静态资源（图片、tools/*.html 等，打包后置于根目录）
├── .next/                # Next.js 编译产物（不加入版本控制）
├── next.config.ts        # Next.js 配置（已启用 Turbopack）
├── tailwind.config.ts    # Tailwind CSS v4 配置
└── tsconfig.json         # TypeScript 配置（@/* 别名 → ./src/*）
```

## 目录使用说明与架构约定

1. **`app/` 薄路由层**：只做路由声明与全局布局，不放任何业务组件。路由文件统一保持 `page.tsx` 全小写，委托给 `src/components/` 中的对应组件处理。

2. **`src/apps/` 微型应用层（App Registry）**：
   - 每个微型应用是独立的 React 组件，统一在 `registry.ts` 注册。
   - 应用通过 `app://<id>` 协议在 `features.ts` 中声明，`FeatureWindow` 自动识别并用 `AppRenderer` 懒加载渲染。
   - **新增应用只需 3 步**：① 写组件 → ② 在 `registry.ts` 注册 → ③ 在 `features.ts` 加 `url: "app://id"` 条目。
   - 按分类组织子目录：`campus/`、`productivity/`、`creative/`、`dev/`。

3. **`src/components/feature-window/`**：功能窗口系统核心，承载三种内容渲染模式：
   - **`app://` 协议** → `AppRenderer` 懒加载 React 组件（推荐用于新建应用）
   - **相对路径** → iframe 内嵌本地 HTML 工具（`public/tools/*.html`）
   - **`https://`** → `window.open` 打开外部链接

4. **`src/data/` 静态数据层**：`features.ts` 是所有功能窗口内容的唯一数据源，`featurePages.ts` 维护各功能页的视觉元数据。纯前端架构阶段不引入数据库，所有状态以静态配置或 `localStorage` 维持。

5. **历史 HTML 工具 (`public/tools/`)**：历史实验性 HTML 文件通过复制至 `public/tools/` 后由 `features.ts` 以相对路径映射，用 iframe 安全渲染，无需重构为 React 组件。
