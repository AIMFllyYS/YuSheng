# 项目详细目录结构规范

本项目基于 Next.js 框架构建，遵循以下目录结构规范：

```text
AIMFlly/
├── app/                  # Next.js App Router 目录，存放页面路由和全局布局
│   ├── favicon.ico       # 网站图标
│   ├── globals.css       # 全局样式文件
│   ├── layout.tsx        # 全局根布局
│   └── page.tsx          # 首页主路由页面
├── src/                  # 核心源代码目录
│   ├── components/       # 可复用组件目录
│   │   ├── effects/      # 视觉效果与动画组件（如粒子、羽毛等特效）
│   │   ├── feature-window/ # 功能窗口组件（如应用导航等）
│   │   └── home/         # 首页专用组件（如底部导航栏、时间显示等）
│   ├── data/             # 静态数据与配置项（如应用列表数据）
│   └── hooks/            # 自定义 React Hooks
├── docs/                 # 项目文档目录
│   ├── source/           # 源码参考或设计文档
│   ├── 规范/             # 项目开发规范（当前目录）
│   └── 其他页面/         # 早期或辅助性设计文件
├── public/               # 静态资源目录（图片、字体等，打包后直接放置于根目录）
├── .next/                # Next.js 编译和构建产生的隐藏目录（不需要加入版本控制）
├── node_modules/         # 项目依赖的 npm 包
├── eslint.config.mjs     # ESLint 配置文件
├── next.config.ts        # Next.js 配置文件
├── package.json          # 依赖配置与项目信息
├── postcss.config.mjs    # PostCSS 配置文件（配合 Tailwind）
├── tailwind.config.ts    # Tailwind CSS 配置文件
└── tsconfig.json         # TypeScript 配置文件
```

## 目录使用说明

1. **`app/` 目录**：只用于路由和页面级别的布局配置。不应将业务组件直接放在此处，应尽量保持轻量。
2. **`src/components/` 目录**：所有的 UI 组件应当存放在此，并按照功能域或业务模块进行分类归档。
3. **`src/data/` 目录**：存放项目中硬编码的配置数据，例如菜单列表、链接集合等，后续若有 API，可作为 mock 数据存放处。
