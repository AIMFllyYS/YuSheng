# 项目详细目录结构规范

本项目基于 Next.js 框架构建，遵循以下目录结构规范：

```text
AIMFlly/
├── app/                  # Next.js App Router 目录，存放页面路由和全局布局
│   ├── favicon.ico       # 网站图标
│   ├── globals.css       # 全局样式文件
│   ├── layout.tsx        # 全局根布局
│   └── page.tsx          # 首页主路由页面
├── src/                  # 核心源代码目录
│   ├── components/       # 可复用组件目录
│   │   ├── effects/      # 视觉效果与动画组件（如粒子、羽毛等特效）
│   │   ├── feature-window/ # 功能菜单与弹窗组件（承接知识库入口、HTML工具箱展示）
│   │   ├── home/         # 首页与全局组件（如人生轨迹Journey、全局功能箱GlobalToolkit）
│   │   ├── notes/        # 富文本与知识库专用组件（为未来 Tiptap/Markdown 预留）
│   │   └── shared/       # 全局通用 UI 组件库
│   ├── data/             # 静态数据与配置项（如功能地图、人生轨迹图节点、静态日历数据）
│   ├── hooks/            # 自定义 React Hooks
│   ├── lib/              # 第三方工具或函数库
│   └── types/            # 全局 TypeScript 类型声明
├── docs/                 # 项目文档与历史遗留资产目录
│   ├── source/           # 源码参考或设计文档
│   ├── 规范/             # 项目开发规范与演进计划（当前目录）
│   └── 单页HTML/         # 历史实验性单页仓库（微积分、三体等，未来通过 iframe 安全渲染）
├── public/               # 静态资源目录（图片、字体等，打包后直接放置于根目录）
├── .next/                # Next.js 编译和构建产生的隐藏目录（不需要加入版本控制）
├── node_modules/         # 项目依赖的 npm 包
├── eslint.config.mjs     # ESLint 配置文件
├── next.config.ts        # Next.js 配置文件
├── package.json          # 依赖配置与项目信息
├── postcss.config.mjs    # PostCSS 配置文件（配合 Tailwind）
├── tailwind.config.ts    # Tailwind CSS 配置文件
└── tsconfig.json         # TypeScript 配置文件
```

## 目录使用说明与架构演进约定

1. **`app/` 目录**：只用于路由和页面级别的布局配置。不应将业务组件直接放在此处，应尽量保持轻量。
2. **`src/components/` 目录**：所有的 UI 组件应当存放在此，并按照功能域或业务模块进行严格分类：
   - `home/`：承载宏大叙事的首页场景（如 `JourneyOverlay`）以及全局级悬浮组件（如 `GlobalToolkit`）。
   - `feature-window/`：作为进入知识库和工具箱的“大门”，负责分类导航与单页 iframe 的调度加载。
3. **`src/data/` 目录**：纯前端架构阶段的核心驱动层。在不使用数据库的前提下，知识库目录树、人生规划图（Roadmap）的进度配置、日历记录等静态数据结构均统一维护于此，确保系统数据流向的一致性。
4. **历史数据沉淀 (`docs/单页HTML/`)**：作为“无数据库”方案的一部分，过往所有高质量的 HTML 交互实验均当作纯静态资源归档，通过前端菜单数据结构（`features.ts`）映射调用，无需进行大规模组件重构即可整合为庞大知识库体系。
