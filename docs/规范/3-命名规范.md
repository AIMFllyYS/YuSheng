# 命名规范

良好的命名规范有助于提升代码可读性与团队协作效率。在本项目中，请严格遵守以下命名规范。

## 1. 文件与文件夹命名

- **React 组件文件**：使用 PascalCase（大驼峰）命名法，例如 `FeatureWindow.tsx`, `BottomNav.tsx`。对于 Next.js 的路由文件（如 `page.tsx`, `layout.tsx`），保持全小写。
- **普通逻辑文件与 Hooks**：使用 camelCase（小驼峰）命名法，例如 `useDraggable.ts`, `formatTime.ts`。
- **样式文件**：与组件同名或使用全小写中划线分隔，如 `globals.css`。
- **文件夹命名**：使用 kebab-case（小写连字符）命名法，例如 `feature-window`, `effects`。组件特有的文件夹可以使用大驼峰。

## 2. 变量与函数命名

- **常规变量与函数**：使用 camelCase（小驼峰）。
  - 变量名应为名词，如 `userInfo`, `activeTab`。
  - 函数名应包含动词，如 `getUserInfo`, `handleSubmit`, `toggleModal`。
- **常量**：使用 UPPER_SNAKE_CASE（全大写下划线分隔）。
  - 例如 `MAX_COUNT = 10`, `API_BASE_URL`。
- **布尔值变量**：建议使用 `is`, `has`, `should`, `can` 等作为前缀。
  - 例如 `isVisible`, `hasError`, `shouldRender`。

## 3. React 组件与 Props 命名

- **组件名称**：使用 PascalCase（大驼峰），与文件名保持一致。
- **Props 接口命名**：组件名加上 `Props` 后缀，如 `FeatureWindowProps`。
- **事件处理 Props**：使用 `on` 作为前缀，如 `onClick`, `onClose`。
- **事件处理函数**：使用 `handle` 作为前缀，如 `handleClose`, `handleClick`。

## 4. CSS 与样式规范 (Tailwind CSS v4)

本项目采用 **Tailwind CSS v4** 作为主要的样式解决方案，遵循 CSS-first 原则：

- **避免任意值**：尽量使用系统内置的值，除非必要不使用任意值（如 `w-[321px]`），转而将其提炼为系统变量或设计 Token。
- **语义化 Token**：对于颜色、间距等，使用语义化命名（如 `--color-primary`）而不是具体的数值名。
- **流式响应式与容器查询**：组件级别的响应式优先使用容器查询（`@container` 与 `@sm:` 等），页面级别的响应式使用常规断点（`sm:`, `md:`）。
- **抽取组件**：当一组工具类被重复使用 3 次以上，或者包含复杂的状态变体时，应抽取为 React 组件，而不是使用传统的 `@apply` 方式。
