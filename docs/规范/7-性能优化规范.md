# 性能优化规范

本文档记录项目已实施的性能优化策略与后续开发必须遵循的性能守则。

---

## 1. 渲染层级模型

项目采用三层渲染架构，各层职责分离，互不阻塞：

| 层级 | 内容 | 渲染方式 | 线程 |
|------|------|---------|------|
| **L0 背景层** | `BackgroundCanvas`（Orb、羽毛、渐变） | Canvas 2D + rAF | 主线程（GPU 加速） |
| **L1 交互层** | Cursor 跟随、弹幕 | CSS transform + DOM 池 | Compositor 线程 |
| **L2 内容层** | Section、FeatureWindow、Overlay | React DOM + CSS Animation | 主线程 |

### 关键规则

- **L0 禁止触发 Layout/Paint**：所有 Canvas 效果仅操作 `<canvas>` 和 CSS `transform`，不得操作 `left/top/width/height`。
- **L1 使用 Compositor 属性**：鼠标光标、弹幕等高频 DOM 更新**只允许**修改 `transform` 和 `opacity`。
- **L2 按需挂载**：`SectionPane` 实现了懒挂载，首次激活前不初始化子组件。新增 Section 必须通过 `SectionPane` 包裹。

## 2. Canvas 性能守则

### 已实施

| 优化项 | 说明 |
|--------|------|
| Orb 渐变缓存 | `createRadialGradient` 仅在 Orb 初始化时调用一次，存入 `orb.gradient`，动画循环中零 GPU 对象创建 |
| 单一指针监听 | 仅注册 `pointermove`（含 `getCoalescedEvents`），不注册冗余的 `mousemove` |
| 羽毛生成节流 | 通过 `lastFeatherTime` 控制最小间隔 150ms，防止高频鼠标移动产生大量粒子 |
| DPR 上限 | `Math.min(devicePixelRatio, 1.75)` 限制高分屏过度渲染 |

### 后续开发必须遵守

1. **禁止在 `animate()` 循环中调用 `createLinearGradient` / `createRadialGradient`**（除非是每帧必须变化的场景，如背景色偏移）。
2. **新增粒子效果必须设置上限**（参考 `MAX_FEATHERS = 40`）。
3. **Canvas 尺寸变化只在 `resize` 事件中处理**，不在动画帧内检测。

## 3. DOM 操作规范

### 已实施

| 优化项 | 说明 |
|--------|------|
| Cursor transform 驱动 | `cursorDot/cursorRing` 使用 `style.transform` 而非 `style.left/top`，消除每帧 4 次 Layout Reflow |
| DanmakuLayer 对象池 | 预分配 6 个 DOM 节点循环复用，零 `createElement`/`remove` |
| glass-card 伪元素阴影 | hover 阴影通过 `::after` + `opacity` 过渡实现，不触发 Paint |
| Journey 拖动 rAF 节流 | `requestAnimationFrame` 批处理拖动回调，DOM 写入上限 60fps |

### 后续开发必须遵守

1. **高频 DOM 更新（>30fps）只允许修改**：`transform`、`opacity`、CSS 变量（`--var`）。
2. **禁止在事件回调中直接同步写多个 DOM 属性**，必须通过 `requestAnimationFrame` 批处理。
3. **新增弹幕/粒子/气泡等频繁创建销毁的元素，必须使用对象池模式**。
4. **`box-shadow` 动画必须使用伪元素 + `opacity` 方案**，而非直接 transition box-shadow。

## 4. 组件懒加载策略

### 已实施

| 组件 | 策略 | 说明 |
|------|------|------|
| `SectionPane` | 首次激活懒挂载 | 非首屏 Section 从未访问则不初始化 JS |
| `AppRenderer` | `React.lazy` + `Suspense` | 微应用按需加载，首屏零体积 |

### 后续开发必须遵守

1. **新增 Section 必须通过 `SectionPane` 组件包裹**，自动享有懒加载。
2. **新增微应用必须在 `AppRenderer` 中使用 `lazy(() => import(...))`**，禁止静态 import。
3. **体积 > 50KB 的第三方库必须动态导入**（如 FullCalendar、ReactFlow 已经是动态引入）。

## 5. 外部资源加载

### 已实施

| 优化项 | 说明 |
|--------|------|
| Preconnect | Google Fonts、Google Static、cdnjs 三个域名预热 |
| Font Awesome 统一 | 全项目图标已迁移至 FA6 (`fas fa-`)，消除 FA4 重复下载 |
| Google Fonts display=swap | 字体异步加载，不阻塞首屏文字渲染 |

### 后续开发必须遵守

1. **新增 CDN 资源必须在 `<head>` 中添加对应的 `<link rel="preconnect">`**。
2. **禁止引入完整图标库**（如 Material Icons 全量 CSS），应按需引入或使用 SVG。
3. **图标类名统一使用 `fas fa-xxx`**（FA6 Solid），不得使用旧版 `fa fa-xxx`。

## 6. CSS 动画性能

### 已实施

| 优化项 | 说明 |
|--------|------|
| page-section 动画缩短 | `0.7s / 30px` → `0.42s / 14px`，减少 glass 合成层重绘时长 |
| transition 精确化 | `transition: all` → 精确列出属性，避免浏览器监听不必要的属性变化 |
| will-change 预声明 | `#journey-viewport`、`#cursor-dot/ring` 提前声明 `will-change: transform` |
| VideoOverlay 单一控制 | 仅通过 CSS class `.show` 控制显隐，移除冲突的 inline style |

### 后续开发必须遵守

1. **禁止使用 `transition: all`**，必须明确列出需要过渡的属性。
2. **`will-change` 只在确实需要的元素上声明**，不得滥用（浏览器会为每个声明分配额外的合成层内存）。
3. **CSS animation 持续时间建议**：微交互 200-300ms，页面切换 300-500ms，装饰性动画可更长。
4. **`backdrop-filter: blur()` 是昂贵操作**，在动画过程中（如 translateY）会导致每帧重新合成。新增 glass 容器在做位移动画时，考虑临时降低 blur 值或使用 `will-change` 提前提升。

## 7. 交互响应优化

### 已实施

| 优化项 | 说明 |
|--------|------|
| 单击即时选中 | `useClickDiscriminator` 在 `onClick` 时立即执行 `selectItem`，0ms 感知延迟 |
| 双击延迟隔离 | 双击打开与单击选中逻辑解耦，互不阻塞 |

### 后续开发必须遵守

1. **视觉反馈必须即时**：用户点击后 ≤100ms 内必须有可见的状态变化（如高亮、缩放）。
2. **长任务提示**：如果某个操作（如加载微应用）需要 >500ms，必须展示 Loading 状态。

---

## 8. 性能基准与验证

### 推荐的验证方法

| 工具 | 用途 |
|------|------|
| Chrome DevTools → Performance | 录制 5s 操作，检查 Long Task（>50ms）和帧率 |
| Lighthouse (Performance) | 首屏 FCP / LCP / TBT / CLS 综合评分 |
| `React Profiler` | 检测不必要的组件 re-render |
| `npm run build` 输出 | 检查各 chunk 体积，关注 >200KB 的模块 |

### 性能红线

| 指标 | 目标 |
|------|------|
| 首屏 FCP | < 1.5s |
| 动画帧率 | ≥ 55fps（Canvas + 页面切换） |
| 主线程 Long Task | 单次 < 80ms |
| 首屏 JS Bundle | < 300KB (gzip) |
